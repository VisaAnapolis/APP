v2.1.2 — 12/02/2026 15:00 (Horário de Brasília – UTC−3)

Sincronização Automática com o CVS a cada 1 hora
  • Integração contínua com o sistema legado (CVS – Delphi/Paradox):
    • Implementado processo automatizado de exportação e sincronização de dados a cada 60 minutos.
    • Atualização incremental de arquivos .CSV e .JSON diretamente do servidor interno.
    • Publicação automática no repositório GitHub (branch main).
  • Pipeline técnico consolidado:
    • Execução automática via script (.bat) com controle de log.
    • git pull → cópia de arquivos atualizados → commit automático → push seguro.
    • Registro detalhado em log para auditoria e rastreabilidade.
  • Atualização quase em tempo real para os fiscais:
    • Novos alvarás, alterações cadastrais e movimentações passam a refletir no VISA App em até 1 hora.
    • Redução drástica da defasagem entre o banco local do CVS e o ambiente web/PWA.
    • Maior confiabilidade para consultas externas e tomadas de decisão.
  • Estabilidade e segurança:
    • Execução sem interface gráfica (background), evitando dependência de login manual no servidor.
    • Controle de erros com verificação de retorno (errorlevel).
    • Evita sobrescrita indevida da branch principal.
  • Marco arquitetural:
    • Consolida o CVS como fonte oficial de dados.
    • Transforma o VISA App em camada de visualização sempre sincronizada.
    • Estabelece base para futuras evoluções (sincronização incremental por checksum, webhooks ou migração futura para banco em nuvem).

Impacto: o VISA App deixa de ser apenas um espelho eventual e passa a operar como painel institucional sincronizado automaticamente com o sistema oficial, elevando o nível de confiabilidade operacional e maturidade tecnológica da Vigilância Sanitária.

Arquivos e Componentes Envolvidos:

  • SyncOnly.bat — rotina automatizada de sincronização.
  • Exportadores CSV/JSON do CVS (Delphi 6).
  • Repositório GitHub (/data).

v2.1.1 — 03/02/2026 23:59 (Horário de Brasília – UTC−3)

Melhorias e Correções na Página de Alvarás
  • Correção do botão “Limpar”:
    • Botão “Limpar” voltou a executar a rotina correta (limparFiltros()), resetando campos e recarregando a listagem.
    • Tratamento defensivo adicional em filtros para evitar falhas quando algum campo vier vazio.
  • Modal (responsivo) — estabilidade e usabilidade no iOS/PWA:
    • Correção do travamento ao tocar fora do modal: fechamento agora é feito por listener no overlay (robusto para touch).
    • Prevenção de propagação de clique/toque no conteúdo do modal para evitar comportamento inconsistente.
    • Melhoria de UX: bloqueio do scroll do fundo enquanto o modal está aberto (body.modal-open).
    • Atalho adicional: tecla ESC fecha o modal no desktop.
  • Atividades (modal) — leitura perfeita no mobile:
    • Correção de colunas “espremidas” (exibiam apenas 1–2 caracteres).
    • Aplicado padrão moderno: quebra de linha + word-wrap, mantendo também scroll horizontal quando necessário.
    • Tabela de atividades passa a usar table-layout: auto e largura mínima para permitir deslize sem perder legibilidade.
  • Coluna "Exercício" (tabela + modal):
    • Adicionada a coluna Exercício como primeira coluna da tabela (ordenável).
    • Campo Exercício incluído no modal de detalhes do alvará.
    • Ordenação numérica correta do exercício (2018 → 2019 → 2020...).
  • Documento "CNPJ/CPF" unificado (regra definitiva):
    • Coluna "CNPJ" renomeada para "CNPJ/CPF".
    • Regra aplicada: se regulado.cgc estiver vazio/null → exibe regulado.cpf.
    • Ordenação da coluna considera o documento normalizado (somente dígitos), mantendo consistência.
    • Exibição com máscara automática (CPF/CNPJ) quando o tamanho é reconhecido.
  • Busca aprimorada (Razão Social + CPF/CNPJ parcial no mesmo input):
    • O campo de busca agora aceita parte do CPF/CNPJ (ex.: “2373” encontra “23.730.770/0001-83”).
    • Busca por documento utiliza comparação por dígitos (includes() em valor normalizado), aceitando entradas sem máscara.
    • Quando o texto contém letras, a busca continua funcionando por Razão Social (como antes).
  • Filtro de qualidade de dados (registros inconsistentes):
    • Descarta registros com exercício < 2018.
    • Descarta registros sem Razão Social preenchida (evita linhas “vazias” e ruído na pesquisa).
    • Validações defensivas para valores nulos/vazios antes de renderizar.
  • Estabilidade visual da tabela (colunas não “pulam”):
    • Correção do efeito de largura variável por ordenação (título “CNPJ/CPF” mudando de posição).
    • Implementado layout estável: table-layout: fixed + larguras fixas para colunas principais.
    • Tratamento de overflow com text-overflow: ellipsis para manter alinhamento.
  • Performance e robustez:
    • Criação de índice em memória para regulados (codigo → regulado) reduzindo buscas repetidas (find) em massa.
    • Cache de regulados/bairros para evitar recomputação e melhorar responsividade.
    • Normalização centralizada de códigos e documentos (remove máscara e espaços) para reduzir inconsistências.
  • Modal e observações:
    • Seção Observações no modal: exibe obs e ob_duam quando preenchidos.
    • Quebra de texto ajustada para evitar estouro de layout em textos longos (white-space: pre-wrap, overflow-wrap).

Impacto: melhora significativa de usabilidade (busca por CPF/CNPJ parcial), consistência visual (colunas estáveis) e qualidade dos registros exibidos (razão social obrigatória), mantendo ordenação e desempenho.

Arquivos Modificados:

  • alvara.html - Página de alvarás (tabela, busca, ordenação, filtros e layout fixo).
