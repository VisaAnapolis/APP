v2.6.0 — 02/04/2026 11:19 (Horário de Brasília – UTC−3)

📋 OS de Auditoria no APP + automação de timestamps de sincronização
  • Novidade no dashboard (index.html): adicionada entrada na seção Novidades no Aplicativo informando que as Ordens de Serviço de Auditoria atribuídas a cada fiscal podem ser consultadas diretamente pelo APP oficial da Vigilância Sanitária de Anápolis. Removida a entrada mais antiga (Busca no Dashboard) para manter o painel enxuto.
  • Novo workflow — update-sync-timestamps.yml (criado): GitHub Actions que dispara automaticamente a cada push no main quando os arquivos data/cnae_aux.csv, data/taxa.csv ou data/auditoria.csv são alterados — ou seja, sempre que o sistema externo (CVS/SIM/OS) faz commit de novos dados. O workflow extrai o timestamp diretamente da mensagem do commit (padrão "Auto-sync data DD/MM/YYYY HH:MM:SS,XX") e grava nos respectivos *_sync.txt (cnae_sync.txt, taxa_sync.txt, auditoria_sync.txt), que são lidos pelo dashboard para exibir a data/hora da última sincronização de cada fonte. Caso a mensagem do commit não contenha esse padrão, usa o timestamp ISO 8601 do próprio commit convertido para America/Sao_Paulo como fallback. Possui guarda anti-loop dupla: condição !contains(github.event.head_commit.message, '[skip ci]') e !startsWith(github.event.head_commit.message, 'Atualizar timestamps'), evitando que o próprio commit do workflow dispare uma nova execução.
  • Correção no workflow — update-sync-timestamps.yml (fix): o passo de git add original usava git add data/cnae_sync.txt data/taxa_sync.txt data/auditoria_sync.txt, que falhava com erro quando algum dos arquivos ainda não existia no repositório (caso do cnae_sync.txt recém-criado). Substituído por um loop condicional for f in ...; do [ -f "$f" ] && git add "$f" || true; done, tornando o git add tolerante à ausência de qualquer arquivo sync. Simultaneamente criado o data/cnae_sync.txt que estava faltante.

Arquivos criados/modificados: index.html — nova novidade OS Auditoria + remoção entrada antiga; .github/workflows/update-sync-timestamps.yml — criado e corrigido; data/cnae_sync.txt — criado; js/version.js — APP_VERSION 2.6.0; service-worker.js — CACHE_NAME visa-v2.6.0; changelog.html — esta entrada.

v2.5.0 — 10/03/2026 12:46 (Horário de Brasília – UTC−3)

✨ Busca Inteligente Unificada — pesquise regulados, OS, protocolos e mais em um único lugar
  • Busca Global no Dashboard: novo campo de busca centralizado no Dashboard (index.html) que pesquisa simultaneamente em 7 fontes de dados — Regulados (CNPJ, razão social, nome fantasia), Ordens de Serviço, Protocolos, Denúncias, Requerimentos, Ofícios e Alvarás. Não precisa mais pular entre páginas para encontrar informações: digite o termo, CNPJ ou número e veja os resultados categorizados instantaneamente.
  • Motor de busca inteligente (js/busca-global.js): módulo dedicado que carrega e indexa os CSVs sob demanda, com normalização de texto (acentos, pontuação, CNPJ com/sem formatação), matching por substring e agrupamento por categoria. Utiliza JOIN otimizado entre fontes para enriquecer resultados de alvarás com dados de regulados em passe único.
  • Interface com botão de ação: a busca é disparada por botão (não por digitação), evitando requisições desnecessárias. Resultados aparecem em painel flutuante com badges coloridos por categoria, contagem de resultados, e links diretos para as páginas de detalhe.
  • Correção de normalização: a função norm() foi corrigida para tratar corretamente buscas por CNPJ sem formatação (ex: 12345678000190), que antes falhavam silenciosamente por remoção excessiva de caracteres.
  • Subtítulo dinâmico atualizado: o subtítulo do Dashboard foi atualizado para refletir o escopo ampliado da página com a busca global.
  • Plantão público: a página plantao.html teve o guard de autenticação removido, tornando-a acessível sem login conforme necessidade operacional.

  • Vinculação imediata após opt-in: quando o usuário aceita notificações, a função initPushNotifications() retorna o token e agora ele é imediatamente salvo em dispositivos[chave].fcmToken via nova função _vincularTokenAoDispositivo(). Tokens excedentes são descartados. Inclui modo dry-run para revisão antes de aplicar.
  • Invalidação de cache alinhada: service worker cache bumped de visa-v2.2.9 para visa-v2.4.9, garantindo sincronismo com versão do app.
