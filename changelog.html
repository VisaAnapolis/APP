v2.5.1 — 10/03/2026 14:20 (Horário de Brasília – UTC−3)

🐛 Correções pós-v2.5.0 — sidebar, permissões e documentação
  • Fix: Links do sidebar bloqueados em páginas com guard: em páginas com autenticação (admin.html, os.html, cvs.html, etc.) que gerenciam seu próprio #userInfo via window.__sidebarUserManaged = true, o sidebar.js agora sempre aplica as permissões de navegação (sidebarAplicarPermissoes(grupo)) mesmo quando a página controla a exibição do usuário. Antes, o callback onAuthStateChanged retornava cedo, deixando todos os links restritos permanentemente desabilitados. Agora: apenas a exibição de nome/email é pulada (respeitando a página), mas as permissões baseadas no grupo (Fiscal, Admin) são sempre aplicadas.
  • Documentação do CLAUDE.md atualizada: adicionadas as características recentes da app: busca-global.js, version.js, header-component.css, tabela completa de 23 páginas com coluna de acesso por grupo, seções detalhadas de diferenças entre guard.js vs guard1.js, permissões de links por sidebar, documentação de busca-global.js como novo módulo chave, atualização da tabela de estado do sidebar colapsável.
  • Alinhamento de versão pendente: ⚠️ APP_VERSION em js/version.js está em 2.5.0 mas cache do service worker em service-worker.js ainda está em v2.4.9 — será bumped na próxima release de UI significativa.

Arquivos modificados: js/sidebar.js — refatoração de onAuthStateChanged para separar "gerenciamento de display" de "aplicação de permissões"; CLAUDE.md — documentação abrangente de estrutura, permissões, versões recentes e plano futuro; changelog.html — esta entrada.

v2.5.0 — 10/03/2026 12:46 (Horário de Brasília – UTC−3)

✨ Busca Inteligente Unificada — pesquise regulados, OS, protocolos e mais em um único lugar
  • Busca Global no Dashboard: novo campo de busca centralizado no Dashboard (index.html) que pesquisa simultaneamente em 7 fontes de dados — Regulados (CNPJ, razão social, nome fantasia), Ordens de Serviço, Protocolos, Denúncias, Requerimentos, Ofícios e Alvarás. Não precisa mais pular entre páginas para encontrar informações: digite o termo, CNPJ ou número e veja os resultados categorizados instantaneamente.
  • Motor de busca inteligente (js/busca-global.js): módulo dedicado que carrega e indexa os CSVs sob demanda, com normalização de texto (acentos, pontuação, CNPJ com/sem formatação), matching por substring e agrupamento por categoria. Utiliza JOIN otimizado entre fontes para enriquecer resultados de alvarás com dados de regulados em passe único.
  • Interface com botão de ação: a busca é disparada por botão (não por digitação), evitando requisições desnecessárias. Resultados aparecem em painel flutuante com badges coloridos por categoria, contagem de resultados, e links diretos para as páginas de detalhe.
  • Correção de normalização: a função norm() foi corrigida para tratar corretamente buscas por CNPJ sem formatação (ex: 12345678000190), que antes falhavam silenciosamente por remoção excessiva de caracteres.
  • Subtítulo dinâmico atualizado: o subtítulo do Dashboard foi atualizado para refletir o escopo ampliado da página com a busca global.
  • Plantão público: a página plantao.html teve o guard de autenticação removido, tornando-a acessível sem login conforme necessidade operacional.

Arquivos modificados: index.html — campo de busca global, integração com busca-global.js, subtítulo atualizado; js/busca-global.js — novo módulo de busca unificada (carregamento de CSVs, indexação, matching, JOIN de alvarás); js/regulados1.js — ajuste na função norm(); plantao.html — remoção do guard de autenticação; protocolo.html — ajustes de integração; alvara.html — ajustes de integração; docs/BUSCA_GLOBAL_UNIFICADA.md — documentação do projeto; changelog.html — esta entrada.

v2.4.9 — 05/03/2026 01:20 (Horário de Brasília – UTC−3)

Vinculação imediata de token FCM ao dispositivo + limpeza de tokens órfãos
  • Vinculação imediata após opt-in: quando o usuário aceita notificações, a função initPushNotifications() retorna o token e agora ele é imediatamente salvo em dispositivos[chave].fcmToken via nova função _vincularTokenAoDispositivo(). Antes o vínculo só era feito na próxima carga de página via obterTokenFCMSilencioso(), que falha em iOS PWA — resultando em dispositivos com ❌ Sem token permanentemente. Este fix elimina o problema para todos os logins futuros.
  • Exibição do token no modal do admin: no painel administrativo, modal Dispositivos agora exibe o início do token (24 caracteres truncados) abaixo de ✅ Ativo, permitindo que o admin confirme visualmente se o vínculo foi feito com sucesso. Ex: cDw3ZEphIDG6-bFB19U2ZB…
  • Limpeza de tokens órfãos no clean.html: nova operação ② Associar tokens órfãos a dispositivos identifica tokens que existem em fcmTokens mas não estão vinculados a nenhum dispositivo, e distribui 1 token por dispositivo sem token, ordenado por ultimoAcesso (mais recente primeiro). Tokens excedentes são descartados. Inclui modo dry-run para revisão antes de aplicar.
  • Invalidação de cache alinhada: service worker cache bumped de visa-v2.2.9 para visa-v2.4.9, garantindo sincronismo com versão do app.

Arquivos modificados: index.html — nova função _vincularTokenAoDispositivo(), chamada após cada initPushNotifications() (3 call sites); admin.html — exibição de token truncado no modal dispositivos; clean.html — nova operação ② de associação de tokens órfãos com dry-run; service-worker.js — CACHE_NAME visa-v2.4.9; js/version.js — APP_VERSION 2.4.9; changelog.html — esta entrada.

v2.4.8 — 05/03/2026 0:30 (Horário de Brasília – UTC−3)

Vinculação de tokens FCM aos dispositivos + exibição de status push no painel admin
  • Token FCM acoplado ao dispositivo: cada registro de dispositivo no Firestore agora carrega o campo fcmToken com o token específico daquele aparelho. Implementado via nova função obterTokenFCMSilencioso(firebaseApp) exportada de js/push-notifications.js, que captura o token silenciosamente (sem solicitar permissão ao usuário) se a permissão já foi concedida. O token é incluído no registroDispositivo antes de gravar no Firestore.
  • Array global de tokens sem duplicatas: a atualização do array fcmTokens na coleção usuarios agora usa arrayUnion (Firestore) para evitar duplicação automática. Garante que o mesmo token nunca aparece duas vezes no array global, mesmo após múltiplos logins do mesmo dispositivo.
  • Status push visual no painel admin: no modal Dispositivos do painel administrativo (admin.html), cada card de dispositivo agora exibe o status de push com ícone e cor: ✅ Ativo (verde) se o dispositivo tem token válido, ou ❌ Sem token (vermelho) caso contrário. Facilita diagnóstico rápido de qual dispositivo tem notificações habilitadas.
  • Invalidação de cache nos dispositivos: service worker cache bumped de visa-v2.2.8 para visa-v2.2.9, garantindo que todos os dispositivos descartam o cache antigo e baixam a nova versão de index.html, admin.html e js/push-notifications.js na próxima visita (mesmo offline, após cache ser revalidado).

Arquivos modificados: js/push-notifications.js — nova função obterTokenFCMSilencioso(); index.html — import arrayUnion, captura token antes de registroDispositivo, adiciona fcmToken ao registro, usa arrayUnion no updateDoc; admin.html — exibição de status push por dispositivo; service-worker.js — cache version v2.2.9; js/version.js — APP_VERSION 2.4.8; changelog.html — esta entrada.

v2.4.5 — 01/03/2026 22:47 (Horário de Brasília – UTC−3)

Correção de header em iOS e ajuste de layout em telas pequenas
  • iOS — header com fundo azul corrigido: a classe .topbar em iOS estava renderizando com background azul em components.css (linha 361, valor hardcoded rgba(44, 90, 160, 0.9)). Corrigido para usar var(--card-bg) que respeita o modo light/dark automaticamente. Adicionado dark mode específico com @media (prefers-color-scheme: dark).
  • Fallback de variável CSS corrigido: o fallback de var(--text-primary) estava definido como #1a3a6b (azul) em vez de #1a202c (cinza escuro). Em iOS Safari com bug de renderização de variáveis CSS em position:sticky, o fallback azul aparecia visível. Corrigido em 6 páginas: sidebar.css, index.html, indicadores.html, inspecoes.html, rmpf.html, total.html.
  • Header sobrepondo conteúdo em telas pequenas (index.html): o topbar estava sobrepondo o conteúdo ao fazer scroll no mobile. Causas: (1) components.css não definia height do .topbar, deixando-o com altura variável; (2) .page-content não tinha padding-top suficiente. Corrigido: adicionado display: flex, min-height: 56px ao .topbar em components.css; adicionado padding-top: 72px !important ao .page-content em mobile para deixar espaço do topbar (52px) + margem.
  • Cascata CSS em index.html: components.css vinha depois do <style> inline no HTML, sobrescrevendo estilos. Utilizados !important para forçar padding-top e altura mínima do topbar em mobile.
  • iOS dark mode padronizado: adicionado @supports (-webkit-touch-callout: none) com fallback #2a2a31 !important em sidebar.css e index.html para garantir que em iOS, quando a variável CSS falha, a cor fica correta (cinza escuro, não azul ou branco).

Afeta páginas: cvs, legislacao, check, changelog, readme, relatorio_plantao_fiscal, cnae, indicadores, inspecoes, rmpf, total e index (dashboard).
