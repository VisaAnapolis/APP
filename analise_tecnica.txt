---------------------------
1. FALTA DE PADRONIZAÇÃO:
   - Cada página tem seu próprio CSS inline para header
   - Estilos duplicados em múltiplas páginas
   - Inconsistências sutis em padding, cores, fontes

2. RECURSOS NÃO UTILIZADOS:
   - header-component.css existe mas não é usado
   - js/header-loader.js existe mas não é usado
   - Sistema de design tokens parcialmente implementado

3. MANUTENÇÃO DIFÍCIL:
   - Qualquer mudança no header requer edição de 30+ arquivos
   - Risco de inconsistências ao fazer alterações
   - Código duplicado dificulta debugging

4. RESPONSIVIDADE:
   - Algumas páginas têm media queries, outras não
   - Comportamento mobile inconsistente
   - Botão OK aparece em todas as resoluções (deveria ser só mobile)

5. DARK MODE:
   - design-tokens.css tem suporte completo
   - Mas páginas com CSS inline não aproveitam
   - Experiência inconsistente entre páginas
